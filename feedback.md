# 🎯 Darts Analysis System Feedback & Backlog

다트 투구 분석 시스템의 완성도를 높이기 위한 개선 사항 및 할 일 목록입니다.

---

### ⚠️ 개발 범위 조정 (Project Scope Revision)
- **최종 목적지 (Final Goal)**: 본 프로젝트의 최종 형태는 **iOS 온디바이스(On-device)** 애플리케이션입니다. 현재 파이썬 환경은 핵심 분석 알고리즘 검증을 위한 프로토타입 단계입니다.
- **하드웨어 연동 제외**: 아두이노 및 센서 연동 기능을 전체 범위에서 제외하고, **소프트웨어/비전 기반 분석**에 집중하여 모바일 환경에 적합한 구조를 만듭니다.
- **LLM 연동 최소화**: LLM(Ollama) 연동은 구조적 형태(Skeleton)만 유지하며, 실제 분석 결과 도출은 모바일 온디바이스에서 빠른 처리가 가능하도록 **룰 엔진 기반 템플릿 피드백**을 우선적으로 사용합니다.

---

### 1. 🔍 분석 알고리즘 고도화 (Core Logic)
- [ ] **투구 횟수 인식 강화**: 특정 동작에서 카운팅이 누락되거나 로직이 멈추는 현상 수정.
- [ ] **릴리즈 타이밍 정밀화**: 다트를 놓는 순간의 팔 각도 및 타이밍 수치화 (횟수별 일관성 체크).
- [ ] **시선 분석 (Gaze Tracking)**: 투구 시 시선이 과녁을 제대로 향하고 있는지 분석.
- [ ] **손동작 분석**: 단순 팔 움직임 외에 손목 및 손가락의 릴리즈 동작 세분화.
- [ ] **다트 폼 유형 분류**: 사용자의 투구 스타일(Standard, Side-arm 등) 자동 판별 기능.
- [ ] **iOS 마이그레이션 대비**: Python/MediaPipe 기반 코드를 iOS Swift/Vision 또는 CoreML 모델로 변환하기 쉬운 구조로 모듈화 유지.

### 2. 📱 UI/UX 및 사용자 가이드 ios ui
- [ ] **리포트 UI 설계**: 분석 결과(각도, 타이밍, 일관성)를 시각화한 대시보드 형태의 리포트 화면.
- [ ] **촬영 가이드라인 표시**: 실시간 화면에 올바른 투구 위치 및 자세 스켈레톤 가이드 오버레이.
- [ ] **촬영 환경 알림**: 
    - 후면 촬영 시 "피드백을 위해 측면/전신이 보이게 촬영하세요" 알림 (후면 인식 불가 이슈 해결).

### 3. ⚙️ 시스템 및 데이터 관리
- [ ] **파일명 문자열 처리 자동화**: `[날짜]_[사용자ID]_[투구차수].mp4` 등 규칙적인 파일 관리 로직 구현.
- [ ] **기준 자세(Reference) 데이터 입력**: 프로 선수나 정석 자세 데이터를 시스템 기준값으로 설정.

### 💡 우선 순위 (Priority)
1. **High**: 촬영 환경 알림 (후면 촬영 제한) 및 투구 횟수 인식 오류 수정.
2. **Medium**: 리포트 UI 개발 및 파일명 처리 자동화.
3. **Low**: 시선 및 손동작 분석 고도화.
